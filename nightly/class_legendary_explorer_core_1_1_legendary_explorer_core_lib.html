◆ SetSynchronizationContext()

static void LegendaryExplorerCore.LegendaryExplorerCoreLib.SetSynchronizationContext ( TaskScheduler  scheduler)
inlinestatic

Allows you to specify the synchronization context, for example, if you want to use Task.ContinueWithOnUIThread() before the library has been initialized.

If your application has a UI thread, you must obtain a task scheduler from the UI thread. For WPF, you can call TaskScheduler.FromCurrentSynchronizationContext from a UI thread, and then pass that object to this method.

Parameters
schedulerTaskScheduler generated from a UI thread. If your application doesn't have a UI thread, passing in any task scheduler should be fine.

loggerSerilog logger to use for logging operations. If null, no logging is performed.
objectDBsToLoadOnly load object info for specified games. If null, all are loaded.
